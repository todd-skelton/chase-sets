# Definition Of Done

Related: `docs/planning/DEFINITION_OF_READY.md`, `docs/engineering/TEST_STRATEGY.md`

A story is done when all are true:

- Acceptance criteria are met and verified.
- Required tests are passing at appropriate levels.
- Contracts/docs are updated (API, events, schema as applicable).
- Observability updates are in place (logs, metrics, traces).
- Security and privacy impacts are reviewed.
- Rollout and rollback plans are documented.
- Feature flags include owner and removal plan.
- Post-merge operational checks are complete.
